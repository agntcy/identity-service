Cross-App Access — Requesting App → Receiving App → Gitea

ID-JAG issuer → Keycloak 26.7 receiver (jwt-bearer) → scoped access token → Gitea via gateway
Requested scope:
The Requesting App gets access to the Receiving App on the user's behalf, then the Receiving App calls Gitea with the exchanged token. The gateway enforces the requested scope: with read-only, listing repos succeeds but creating one is denied (403) — narrow scoping in action.
